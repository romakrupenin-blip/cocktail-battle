# Cocktail Battle — ТВ + ведущий (Cloudflare Workers)

Бесплатный сервер на Cloudflare Workers, который синхронизирует экран телевизора
(только показ) и телефон ведущего (весь ввод) в реальном времени через WebSocket.

## Что где

- `/tv`   — открыть на телевизоре. При каждом заходе создаётся новая комната
            и показывается QR-код с ссылкой на `/host`.
- `/host` — открывается по QR ведущим. Тот же интерфейс, что и в телефонной
            версии (участники, фото, звёзды), только данные синхронизируются
            с ТВ через сервер.

## Деплой (один раз)

Нужен Node.js (npm) на компьютере или self-hosted сервере.

```bash
npm install
npx wrangler login      # откроется браузер, войти в бесплатный аккаунт Cloudflare
npx wrangler deploy
```

После `deploy` в терминале появится ссылка вида:

```
https://cocktail-battle.<твой-поддомен>.workers.dev
```

Это постоянный адрес. На телевизоре открываешь:

```
https://cocktail-battle.<твой-поддомен>.workers.dev/tv
```

Всё, дальше — сканируешь QR телефоном и пользуешься как обычно.

## Локальная проверка перед деплоем (по желанию)

```bash
npm run dev
```

Откроется локальный адрес (обычно `http://localhost:8787`) — можно проверить
`/tv` и `/host` на одном компьютере в двух вкладках, прежде чем выкладывать
в интернет.

## Тесты логики

```bash
npm test
```

Прогоняет проверку основной логики (старт, оценивание, блокировка результатов,
новая вечеринка) без необходимости поднимать сервер.

## Важные технические детали

- Состояние вечеринки хранится в Durable Object (SQLite-backend, входит в
  бесплатный тариф Cloudflare Workers).
- Менять состояние может только подключение с `role=host` — экран ТВ всегда
  read-only, даже если знать URL комнаты.
- Переход к результатам физически заблокирован на сервере, пока не оценены
  все коктейли всеми участниками (не только на клиенте — подделать нельзя).
- Фото сжимаются на телефоне перед отправкой (макс. 480px, JPEG), чтобы не
  раздувать трафик и хранилище.
